## 2. 核心玩法介绍

### 2.1 玩家目标

玩家驾驶战机在 800×600 的战场中以俯视角射击不断涌来的敌方编队，通过击杀敌人获取经验值升级、分配技能点构建专属流派，在无尽攀升的难度中追求最高分。

### 2.2 游戏基本规则

| 参数 | 值 | 说明 |
|------|-----|------|
| 初始生命值 | 5 | 菱形血条图标，受击减少 |
| 最大生命值 | 5 | 可通过 heal 道具回满 |
| 受击无敌时间 | 1.5 秒 | 被击中后短暂无敌，角色闪烁提示 |
| 最高分持久化 | `localStorage` | 仅保存历史最高分，不保存其他进度 |
| 游戏状态 | 内存中 | 关闭页面后所有进度丢失 |

### 2.3 难度递进机制

游戏采用**时间驱动难度**系统，难度值 `difficulty = 1 + totalTime / 30`：

| 游戏时间 | 难度值 | 敌人生成间隔 | 出现的新敌人 |
|----------|--------|-------------|-------------|
| 0 秒 | 1.0 | 1.20 秒/波 | 战斗机、方形、三角形 |
| ~18 秒 | 1.6 | 0.75 秒/波 | + 菱形、炮塔 |
| ~36 秒 | 2.2 | 0.55 秒/波 | + 轰炸机 |
| ~45 秒 | 2.5 | 0.48 秒/波 | + 中型 Boss（首次） |
| 持续增长 | — | 最低 0.25 秒/波 | 所有敌人类型 |

### 2.5 主动技能系统

玩家拥有 4 个技能槽位（含默认普攻），3 个主动技能需在技能树中解锁：

| 槽位 | 按键 | 技能名称 | 解锁等级 | 冷却时间 | 效果描述 |
|------|------|----------|---------|---------|---------|
| 0 | 空格 / Z / J | **等离子主炮** | Lv1（默认） | 无 | 基础射击。随等级自动强化：Lv5 +30% 伤害、Lv10 双发、Lv20 三发、Lv30 穿透、Lv45 灼烧 |
| 1 | 1 | **导弹齐射** | Lv3 | 8 秒 | 扇形发射 3+ 枚追踪导弹，高伤害穿透 |
| 2 | 2 | **持续激光** | Lv6 | 10 秒 | 头顶发射持续光束（3 秒），可消弹、穿透 |
| 3 | 3 | **无人机集群** | Lv15 | 15 秒 | 召唤环绕无人机持续自动输出（10+ 秒） |

> **零CD 道具效果**：拾取后所有技能忽略冷却限制，持续 5 秒。

### 2.6 技能树系统

#### 2.6.1 概述

游戏采用**类暗黑破坏神 4 技能树**设计，包含 3 大学派、4 层节点、共 69 个节点：

- **3 大学派**：追踪导弹（橙色）、激光（黄色）、无人机（粉色）
- **4 层结构**：
  - **T0 基础层**（1 节点/学派）：解锁对应主动技能
  - **T1 核心层**（3 节点/学派）：数值增强，最多 5 阶
  - **T2 进阶层**（2 节点/学派）：新增行为，最多 3 阶
  - **T3 关键被动**（1 节点/学派）：定义流派核心，仅 1 阶

#### 2.6.2 关键被动互斥机制

每个学派仅有一个 T3 关键被动，**全局只能激活 1 个**。切换关键被动时需确认对话框，防止误操作。三个关键被动分别为：

| 学派 | 被动名称 | 效果 |
|------|----------|------|
| 导弹 | 🔑 饱和覆盖 | 击杀叠层，下次齐射 +6 导弹（持续 5 秒） |
| 激光 | 🔑 湮灭激光 | 持续命中同一敌人，每 0.5 秒 +25% 伤害（最多叠 8 层） |
| 无人机 | 🔑 永恒机群 | 无人机持续时间 → 永久（但数量锁定 2 架） |

### 2.7 巅峰系统（Paragon）

#### 2.7.1 解锁条件

```

```

| 阶段 | 等级范围 | 奖励 | 说明 |

巅峰「幸运币」节点提供额外免死机制：死亡时有 10/20/30% 概率自动复活（不消耗复活币），复活后获得 2 秒无敌时间。

### 2.11 碰撞检测系统

所有碰撞均采用**圆形碰撞检测**（欧氏距离平方比较），碰撞半径取实体自身 radius 属性：

| 碰撞对 | 处理逻辑 |
|--------|---------|
| 玩家子弹 vs 敌人 | 子弹消失 → 敌人扣血 → 击杀时加分/经验/爆炸粒子 |
| 敌人子弹 vs 玩家 | 子弹消失 → 玩家受伤（无敌时间内不触发） |
| 敌人本体 vs 玩家 | 玩家受伤；非 Boss 敌人碰撞后销毁；Boss 不销毁 |
| 玩家 vs 道具 | 道具消失 → 应用效果 |
| 激光束 vs 敌人 | 持续伤害 → 可摧毁敌方子弹 |
| 导弹分裂弹 vs 敌人 | 微弹命中后消失 → 敌人扣血 |

## 3. 操作指南

### 3.1 按键映射

| 按键 | 功能 | 备注 |
|------|------|------|
| ↑ ↓ ← → | 移动战机 | 仅支持方向键移动，不支持 WASD |
| 空格 / Z / J | 基础射击（等离子主炮） | 三个按键均可触发 |
| 1 | 释放技能：导弹齐射 | 需 Lv3 解锁 + 技能树加点 |
| 2 | 释放技能：持续激光 | 需 Lv6 解锁 + 技能树加点 |
| 3 | 释放技能：无人机集群 | 需 Lv15 解锁 + 技能树加点 |
| C | 打开/关闭技能树面板 | 打开时游戏自动暂停 |
| Escape | 关闭技能树面板 | 仅在技能树打开时有效 |

> **注意**：方向键为唯一移动输入，无法使用 WASD 移动。对角线移动时速度已归一化。

### 3.2 游戏流程

```
┌─────────┐    开始游戏     ┌─────────┐    按 C 键     ┌─────────┐
│  开始    │ ──────────→  │  游戏中  │ ──────────→  │  暂停    │
│  画面    │              │         │              │(技能树) │
└─────────┘              └────┬────┘              └────┬────┘
                              │                        │
                         生命值归零              关闭技能树
                              │                        │
                              ▼                        │
                         ┌─────────┐                   │
                         │ 游戏结束 │                   │
                         └────┬────┘                   │
                              │                        │
                    ┌─────────┼─────────┐              │
                    ▼                   ▼              │
              使用复活币          重新开始              │
                    │                   │              │
                    └───────→ 游戏中 ←──┘              │
                                                   恢复游戏 ←┘
```

### 3.3 游戏流程限制

| 限制 | 说明 |
|------|------|
| 移动限制 | 战机限制在画布内，四周留 16px 边距 |
| 技能树操作 | 仅在暂停状态下可操作技能树（按 C 暂停） |
| 暂停冻结 | 暂停时所有游戏逻辑停止（敌人/子弹/粒子/计时器全部冻结），画面保持最后一帧 |
| 技能冷却 | 主动技能释放后进入冷却，零CD 道具期间例外 |
| 护盾消耗 | 护盾每次受击消耗 1 层充能，充能耗尽或计时结束则护盾消失 |
| 连射叠加 | 连射道具最多叠加 6 层，每层额外 +30% 射速 |

## 4. AI 技术应用说明

### 4.1 项目中的 AI 使用概况

霓虹战机项目**不包含运行时 AI 模型推理**。游戏所有逻辑（敌人行为、碰撞检测、道具掉落、技能计算）均为确定性规则驱动，由 JavaScript 在浏览器中实时执行。

AI 在本项目中的应用局限于**开发阶段的辅助编码**，而非游戏内的智能系统。

### 4.2 AI 辅助开发工具链

#### 4.2.1 MetaGPT 集成

项目构建工具链中集成了 **MetaGPT** 平台相关依赖：

| 依赖包 | 用途 |
|--------|------|
| `@metagptx/vite-plugin-source-locator` | Vite 构建插件，用于源码定位 |
| `@metagptx/web-sdk` | Web SDK，提供客户端 API 集成能力 |

**涉及文件**：

```
app/frontend/
├── vite.config.ts          # 引入 viteSourceLocator 和 atoms 插件
├── package.json            # 声明 @metagptx 依赖
├── pnpm-workspace.yaml     # 工作区配置
└── src/lib/api.ts          # 导入 createClient
```

**实际使用方式**：

- `viteSourceLocator`：在 Vite 开发服务器中注入源码映射信息，便于 AI 辅助工具（如 MetaGPT Atoms 平台）精确定位代码上下文，用于代码审查和智能编辑。
- `atoms` 插件：为 MetaGPT 的 Atoms 文档系统（`.atoms/` 目录下的结构化文档）提供构建时集成。
- `createClient`：提供与 MetaGPT 后端通信的客户端 SDK（当前项目中 `api.ts` 已声明但未在游戏逻辑中实际调用）。

#### 4.2.2 MetaGPT Atoms 文档系统

项目根目录 `.atoms/` 包含以下 AI 辅助文档，用于指导 AI 理解项目上下文：

| 文档 | 用途 |
|------|------|
| `ARCHITECTURE.md` | 架构设计文档，描述系统概览、模块划分、技术决策 |
| `ATOMS.md` | 项目背景与约束条件，记录关键设计决策历史 |
| `PROGRESS.md` | 需求与进度追踪，包含用户故事、任务分解、进度日志 |

这些文档为 AI 辅助工具提供结构化的项目知识，使 AI 能够在正确的上下文中进行代码生成、修改和审查。

### 4.3 游戏内"AI"——敌人行为系统技术说明

虽然项目不使用 AI/ML 模型，但游戏中的敌人行为系统采用了**基于规则的有限状态机**设计，值得从技术角度说明。

#### 4.3.1 敌人行为架构

```
┌─────────────────────────────────────────────┐
│              updateEnemy() 主循环              │
│                                             │
│  ┌─────────┐    ┌──────────┐    ┌────────┐  │
│  │ 移动阶段 │ →  │ 阶段判定  │ →  │ 行为执行 │ │
│  │ (Move)  │    │ (Phase)  │    │ (Act)  │  │
│  └─────────┘    └──────────┘    └────────┘  │
│       │              │              │        │
│   速度向量       状态切换        弹幕生成     │
│   边界检测       haltY 到达      冷却管理     │
│   正弦波动       生命阈值        瞄准计算     │
└─────────────────────────────────────────────┘
```

#### 4.3.2 行为实现模式

每种敌人类型通过以下参数矩阵定义行为：

```typescript
type Enemy = Entity & {
  kind: EnemyKind;      // 类型标识
  score: number;        // 击杀得分
  hp: number;           // 当前生命值
  maxHp: number;        // 最大生命值
  spawnTime: number;    // 生成时间戳
  fireCooldown: number; // 射击冷却计时器
  burstCount: number;   // 连射计数
  phaseTime: number;    // 行为阶段计时器
  baseX: number;        // 正弦波中心 X
  amplitude: number;    // 正弦波振幅
  frequency: number;    // 正弦波频率
  haltY?: number;       // 停留高度（炮塔/炮舰/Boss）
  angle: number;        // 旋转角度（弹幕旋转用）
};
```

#### 4.3.3 弹幕生成算法

| 弹幕类型 | 算法描述 |
|----------|---------|
| **环形弹幕** | 以敌人位置为圆心，按 `2π / count` 等分角度发射 N 发子弹，角度随时间以恒定角速度旋转 |
| **扇形瞄准弹** | 计算敌人→玩家的向量角作为中心线，按等间距偏移发射 N 发子弹 |
| **散弹** | 以敌人正下方为中心，按 `spreadAngle / (count - 1)` 等分展开发射 N 发低速大半径子弹 |

#### 4.3.4 碰撞检测算法

所有碰撞使用**距离平方比较**（避免昂贵的 `Math.sqrt` 运算）：

```typescript
// 伪代码
const dx = a.x - b.x;
const dy = a.y - b.y;
const distSq = dx * dx + dy * dy;
const hitDist = a.radius + b.radius;
if (distSq < hitDist * hitDist) { /* 碰撞发生 */ }
```

激光束碰撞采用 **AABB + 垂直距离** 混合检测：
- 主光束：检查敌人 X 坐标是否在光束宽度范围内，且 Y 在光束上方
- 棱镜分束：将分束方向投影到敌人的相对位置，计算垂直距离

#### 4.4.2 潜在 AI 扩展方向

如需在游戏中引入真正的 AI 功能，以下是推荐的集成点：

| 方向 | 实现方式 | 集成点 |
|------|---------|--------|
| **自适应难度** | 基于玩家表现（命中率/生存时间/得分速率）动态调整生成参数 | `GameCanvas.tsx` 中的 `spawnEnemy()` 和 `difficulty` 计算 |
| **智能敌人行为** | 引入行为树或强化学习 agent 替代固定规则 | `updateEnemy()` 中的各类型分支逻辑 |
| **内容生成** | LLM 生成技能描述、成就文案、剧情对话 | `skills.ts` 中的 `name`/`short`/`description` 字段 |
| **平衡性分析** | 模拟数万局游戏统计各流派胜率/通关时间 | 独立分析脚本，读取 `skills.ts` 和 `constants.ts` 参数 |

> **注意**：以上均为潜在扩展方向，当前版本未实现。
